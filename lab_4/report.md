### Problem:

Это создаёт обширное поле для телефонных мошенников, так как(статья) зная дату рождения, электронную почту, телефон и расхешированный номер карты можно воспользоваться социальной инженерией. Притвориться родственником, свести с ума чьют-то бабушку или по ответам на телефоне с помощью нейросети скопировать голос владельца этой карты. Всё это приводит нас к угрозе состояния средств 48.387.008(статья) пользователей.

## How does the program work?:

Приложение имеет три режима работы.

### cor:

Пример его применения можно найти в файле[lab_4/Solutions/moon/commands.txt]. У нас имеется номер карты и с помощью алгоритма луны мы проверяем корректен ли он написан.

Есть несколько дополнений, которые расширяют его функционал и делают удобным для использования. 

Первое: вместо указания конкретной карты мы можем задать целый пулл карт в виде файла[lab_4/Solutions/moon/cards.txt], который будет анализировать файлы построчно, после в результате работы вернёт файл[lab_4/Solutions/moon/output_correct.json].

Второе: в самой программе имеется список разделителей, которыми по дефолту являются {",", " ", ".", "-"}. Если найдётся символ, не являющийся сепаратором или цифрой, программа пометит этот номер карты "fail input", а в логах[lab_4/Solutions/moon/correct.log] выведется пометка @warning, при этом не завершая работу программы

### Different operating mode gen:

Этого режима работы на мой вкус быть не должно, однако это альтернатива parser.

Этот режим генерирует бины сбербанка, а список бинов построчно можно увидеть в файле[lab_4/Solutions/rehash/bins_default.txt]. А команду в файле[lab_4/Solutions/rehash/commands.txt]/

### Different operating mode un:

Это основной функционал программы - рехеширование с помощью функии blake2s. В начале мы указываем bin банка, а в самом конце последние 4 цифры. Бин задаётся с помощью файла, который мы могли получить в режиме работы gen или создаётся собственный файл[lab_4/Solutions/rehash/bins_parser.txt]. 

Примеры в файле[lab_4/Solutions/rehash/commands.txt].

Результат работы содержится в файле[lab_4/Solutions/rehash/output_default_bins.json].

Важно упомянуть, что имеется команда -cnf для вывода конфигурации в файл вывода, чтобы была возможность проанализировать что именно было использовано для рехеша этого файла.

### Different operating mode dr:

Тут всё просто. Используется функционал команды un, но эта команда просто покажет графики[lab_4/Solutions/draw/output.png], для определения оптимального числа ядер для ускорения работы программы. Число ядер задано по умолчанию, однако можно задать его самостоятельно с помощью флага -mp 24. В данном режиме он используется как максимальное число ядер(для теста и проверки сколько ядер даст наивысшую скорость), а в других режимах указывает конкретное число ядер для работы.
